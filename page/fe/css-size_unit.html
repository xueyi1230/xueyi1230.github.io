css尺寸单位

CSS中常用的有三类：
相对单位：px、ex、em
绝对单位：
百分比：%

W3C推荐：
px（pixel - picture element）像素
相对于屏幕分辨率的字体大小
整数（设备像素比devicePixelRatio >= 2时可设置值为.5，否则将识别为0）
px是绝对单位，不支持IE的缩放。
通过浏览器字体大小设置无效（chrome/IE/firefox）
eg：http://xueyi1230.github.io/page/others/chat.html

最常用，不灵活
em（font size of the element）
相对于当前对象内文本的字体大小（高度）
可指定到小数点后三位
通过浏览器字体大小设置可改变字体大小（firefox无效）
eg：http://v1.jontangerine.com/silo/css/elastic-layout/

计算方法复杂：当前元素字体大小使用相对于父元素计算；当前元素行高/尺寸相对于自身字体大小计算

em最初表示的是字体中大写M的宽度及所用的尺寸，ex？字体x的高度
rem（font size of the root element）
相对于
IE9+
 

使用等宽字体（如<pre>默认font-family: monospace）时，em/rem均会产生奇怪的效果

使用方法：
1）62.5
firefox/IE正常，chrome因为不支持中文浏览器（charset=UTF-8）字体大小小于12px，因此计算不准确
2）625

PPI/DPI（pixels per inch/device pixels per inch）
对角线像素，除以屏幕尺寸（对角线长度）
eg：iphone 6s plus 
计算=400.528…
（分辨率：显示分辨率，单位像素；像素密度：单位像素每英寸，与图片分辨率相同）

不同密度区间，对应不同默认缩放比
xxhdpi480，xxxhdpi640
viemport的target-densitydpi属性，默认值为medium-dpi（160）

iphone4（640*960）首推retina屏幕，320成为超高密度的分界线

回来看devicePixelRatio：
设备像素比devicePixelRatio，是指设备上物理像素和设备独立像素dip(device-independent pixels )的比例
其中，物理像素，就是iphone4的640*960；
          设备独立像素dip，是基于设备屏幕密度的抽象单位，换算公式为dip = 160 * px / density （px = dip * density / 160）
对于iphone4的宽度尺寸，dip = 160 * 640 / 320 = 320
devicePixelRatio = 640 / 320 = 2
以此可以来区分视网膜设备还是非视网膜设备（0.5px）

当设置<meta name="viewport" content="width=device-width">时，iphone4的视区宽度并不是640像素，而是320像素

rem方案被证实在不同手机上显示不一致：http://f2e.souche.com/blog/yi-dong-duan-zi-gua-ying-fang-an/#clue
